# Library of Things (LoT)

## Description

A **community‑owned inventory** of durable goods (tools, appliances, vehicles, event equipment) that members borrow rather than buy. LoTs reduce private ownership overhead, increase access, and build relational wealth.

## Governance

- Each LoT is managed by one or more Guilds via CivicNotes.
- Guild members propose acquisitions, set borrowing rules, and schedule maintenance.
- Decisions are transparent and subject to community review.

## Economic Integration

- Borrowing can be tracked with **Φ‑currency** (small demurrage fees for long‑term holds) or **mutual credit hours**.
- Usage data feeds the Φ‑Grid, contributing to the “Resource Flow” sub‑metric.

## Example User Flow

1. A Guild proposes a LoT via a CivicNotes petition.
2. Community votes; if approved, the Guild acquires assets (using mutual credit or grant funds).
3. Residents browse the LoT catalog, reserve items, and pick them up.
4. Borrowing activity is recorded; frequent use triggers Proof‑of‑Regeneration minting (see PoR component).

## Technical Implementation

- **Inventory database** with availability status, reservation system.
- **Smart contract** (optional) for collateralized borrowing using Φ‑currency.
- **Integration** with CivicNotes to display LoT usage in Guild dashboards.
